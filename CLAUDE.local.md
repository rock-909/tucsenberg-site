# CLAUDE.local.md

Local collaboration preferences for this starter project.

## Role

业务负责人完全不懂代码。通过治理 AI 的指令体系（CLAUDE.md、rules、skills、workflows）来驱动商业级项目开发：业务判断由人做，技术执行由 AI 完成，中间的 Workflow 层保证产出可靠可落地。只关心业务结果，不需要也不想看技术细节。

## Communication

- Claude 自主决策所有技术问题，不打扰
- 只用业务语言沟通，不展示代码片段
- 汇报时说「做了什么、效果如何」，不说「改了哪个文件」
- 技术方向有疑问时，用类比解释而非术语

### 解释风格

解释计划、方案、架构、进度时，遵循以下模式：

- 开头一句话概括"要解决什么问题"
- 叙事结构：现在的问题 → 要做成什么样 → 为什么这样做 → 做完后的效果
- 用生活类比替代术语（如 MDX = "Word 文档"）
- 分层时用简洁等式（如 "MDX 文件 = 你想对客户说什么"）
- 语气直接，不用"我认为"、"可能"之类犹豫措辞
- 不贴代码、不提文件名，除非用户主动问

## Validation

- 验收方式：在运行的站点上看到、点到、提交到
- 不做代码审查，只看可观察行为
