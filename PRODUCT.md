# Product Context

## Status

这是 `showcase-website-starter` 的项目级产品上下文文件，给设计、内容、实现类 AI agent 读，作为本仓项目真相。

本仓是一个**可复用的 starter**，不是任何真实客户的成品网站。所有示例公司名、产品、客户案例、联系方式都必须在用于真实项目前替换掉。

## Register

brand

> showcase 网站属于品牌/营销面：hero、产品故事、信任证据、转化路径——设计本身就是产品，不是辅助某个内部工具。所以 impeccable 的 brand 分册规则适用。

## Users

starter 的两类用户必须同时被服务好。

### 最终访客（网站本身的目标受众）

默认优先服务 **外贸 B2B / OEM/ODM / 制造服务一体化** 型企业，但不能把结构写死成某一个细分行业。

| 角色 | 关心什么 | 网站要给的答案 |
| --- | --- | --- |
| 采购 / 供应链经理 | 这家工厂能不能做我要的东西，按时按质交付 | 产品/服务范围、产能、案例、响应路径 |
| 技术评估方（买方工程师） | 规格、工艺、合规、定制空间 | 具体规格表、工艺细节、标准认证、定制流程 |
| 项目负责人 / 决策者 | 这家公司够不够靠谱跟一年以上 | 公司沉淀、长期客户、组织规模、可验证的细节 |
| 渠道伙伴 / 分销商 | 商务条件、响应速度、可不可预期 | 合作模式、响应承诺、过往合作证据 |

### Starter 的二次使用者（owner / agent）

- 想用这个底座做自己网站的**非技术业主**，希望照着替换清单快速完成第一版换皮，不需要先理解代码结构。
- AI agent 负责具体执行，所以替换面必须明显、文档化、不藏在组件里。

## Product Purpose

默认优化的派生场景：帮一家"制造能力 + 工程服务"型公司（典型：OEM/ODM、定制集成、专业制造商）做一个**可信、不喧哗、转化路径清晰**的官网。

成功的样子：

1. 访客打开首页 10 秒内知道"这家公司做什么、值不值得继续看"。
2. 访客在三屏之内能找到"和我的需求有关的具体证据"。
3. 询盘表单是访客**自然走到**的下一步，不是被弹窗逼出来的。
4. 业主把这个底座拿去换皮的时候，不需要技术人员，照着替换清单就能完成。

## Brand Personality

**三个词：专业、当代、可靠。**

声音：

- 直接、具体、克制。
- 用事实和细节说话，不堆形容词。
- 像一个做了十几年这行的工程总监跟客户讲话——不是销售腔，不是技术宅，是"我知道你真正在乎什么"的那种。
- 中英文都要"专业人讲专业话"的气质，不能让任何一边读起来像翻译稿。

## Anti-references

### 主反例：Alibaba.com 式 B2B 供应商网站

owner 明确点名要躲开的四个毒点：

1. **视觉密度过高**——首页堆满产品瓦片、供应商方阵，眼睛不知道看哪里。
2. **信任靠徽章堆**——金色徽章、年限、认证图标、"诚信通"式装饰物当证据。
3. **没有视觉主角**——任何供应商看起来都一样，没人记得住谁是谁。
4. **模板感重**——一眼就能识别出建站平台批量生产的痕迹。

starter **必须走反方向**：

- 留白多，一屏讲清一件事。
- 信任靠具体案例、真实数字、可验证的工艺照片，不靠徽章。
- 有强烈的视觉主角（hero、关键产品特写、工艺特写）。
- 看起来像是专门为这家公司做的，不像模板拖出来的。

### 其他要躲开的

- **千篇一律 SaaS 模板**：巨型渐变 + 三行假大空标语 + 同样的功能卡片网格。
- **假冒高端**：黑金配色配空洞内容，用 stock 商务握手图硬撑。
- **硬件公司放飞自我**：满屏 3D 渲染但找不到联系方式。

## Design Principles

1. **证据贴着主张走**——每一条卖点旁边都要有可验证的细节（数字、客户名、规格、工艺照片）。不让访客做"信任跳跃"。
2. **节奏胜过密度**——宁可一屏只讲一件事。Alibaba 的反面就是节奏感。
3. **可替换性是一等公民**——所有品牌专属内容（公司名、产品、图片、配色）必须走可见的替换面，不能埋进组件代码里。
4. **多语言是平行内容源**——中英文都是一等公民，谁也不是谁的副本。
5. **工艺感来自细节**——留白、对齐、字号阶梯、动效曲线这些"匠人感"才是让访客觉得"这家不一样"的真正来源，不是大特效。

## Accessibility & Inclusion

- **WCAG 2.2 AA** 作为默认标准。
- 所有可交互元素必须有可见焦点状态。
- 颜色不能是唯一的状态指示（必须有图标、文字、形状辅助）。
- `prefers-reduced-motion` 必须生效。
- 中英文移动端正文字号 ≥16px，行高 ≥1.5。
- 关键路径在禁用 JavaScript 时必须仍有基本 HTML 体验：主页、导航、语言 fallback、联系页表单结构要可读。当前询盘真实提交默认依赖 JS + Turnstile；如果派生项目要求 no-JS 可提交，需要作为单独实现和 E2E 证明处理。
