## Findings considered and rejected (do not re-audit)

- **Production `NEXT_PUBLIC_BASE_URL` placeholder (`example.invalid`) in
  `wrangler.jsonc:44-45`**: NOT a gap. The deploy workflow
  (`.github/workflows/cloudflare-deploy.yml:133-136`) runs
  `PUBLIC_LAUNCH_STRICT=true … validate-production-config` for production, and
  `scripts/quality/checks/production-config.js` (`containsStarterMarker`,
  `validateWranglerProductionPublicUrls`) hard-fails on `example.invalid`.
  This is a deliberate launch gate until the real domain is signed off. Only
  residual action: fill the real domain at sign-off (business milestone, not a
  code plan). A manual `opennextjs-cloudflare deploy` outside CI would bypass
  the gate — accepted; CI is the documented deploy path.
- **`defineSiteDefinition` identity wrapper (`src/config/site-definition-builder.ts`)**:
  kept. Its `const` type parameter provides literal-type inference for
  `SINGLE_SITE_DEFINITION`; replacing with `as const satisfies` risks
  readonly-vs-mutable type churn for zero user value. 7 harmless lines.
- **Product diagram components without tests** (`boxwall-cross-section.tsx`,
  `factory-pool-diagram.tsx`, `product-diagrams.tsx`): presentational SVG, no
  data/mutation logic, covered by page-level Playwright smoke. Not worth
  dedicated tests.
- **`airtable` SDK dormancy (0.12.2 is latest, Node-8-era types)**: already
  isolated behind `src/lib/airtable/**` (`instance.ts` is the sole
  instantiation, marked serverExternal in `next.config.ts`). Keep the wrapper
  as the only call site; a future move to Airtable's REST API stays localized.
  No action now.
- **Performance suspicions all verified settled or clean**: global CSS floor,
  client-messages breadth, shared motion/layout client floor, Cloudflare
  `unstable_cache` bypass, middleware→proxy rename deferral — all documented
  decisions in `docs/技术难题/**` / `.claude/rules/cloudflare.md`;
  `airtable`/`resend`/`@react-email` are server-only (no client-bundle leak);
  no MDX re-parsing at runtime (pre-generated manifest); catalog lookups are
  trivial in-memory scans. `pnpm audit --prod` clean; no major-version lag
  (lucide-react 1.x is current major).
- **Lead-pipeline unit tests "lock old behavior"**: re-judged after the owner
  decision — they lock the *canonical* behavior; plan 001 aligns the docs
  instead of the tests.
