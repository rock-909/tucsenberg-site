## 资产一:TB-BW Boxwall 自锚固剖面动画

### 一句话概念

一张**工程剖面图**:洪水被一块 **L 形挡水板**挡住,而挡水板不靠螺丝固定在建筑上——
是**被它拦下的洪水,用水的自重把底板压在地面上**,越淹越稳。水面轻微起伏(动),其余静止。

### 内核(不可丢)

把 TB-BW 最核心、却最抽象的卖点——**freestanding / tool-free / no fixing to the building
(自立、免工具、不打孔上墙)**——变成一眼自明的物理画面:买家不读字也懂"为什么不固定还扛得住水"。
**"用洪水自身重量自锚固"这个叙事是整图的灵魂**,迁移时其它都可调,这点不能改。

### 画面构成(从下往上、从左到右)

一个带**页眉 + 画面 + 页脚注**的卡片框(像一张仪器面板):

1. **页眉**:等宽小字标签 `TB-BW BOXWALL · SELF-ANCHORING PRINCIPLE` + 右侧一个小状态点。
2. **左侧竖直刻度尺**:`600 / 450 / 300 / 150 / 0`(mm),代表挡水"水头高度"——强调这是有量纲的工程图。
3. **极淡水平网格线**(几条),工程图底纹。
4. **地面基线**(底部实线),线下方画**斜向排线(hatching)**表示土壤。
5. **L 形挡水板(主角)**:
   - 右侧一道**竖直墙体**,高过水面;
   - 墙体底部向**左**伸出**楔形水平底板(apron)**,贴地、泡在水下,近墙处厚、向左渐薄到尖;
   - 墙顶一道**强调色顶帽**(细),旁标 `TB-BW`。
6. **水(洪水侧,墙的左边)**:淡蓝填充到"设计水头"高度;水面为**起伏曲线**;水头处一条**虚线参考线**。
7. **向下荷载箭头(LOAD)**:底板正上方一排 4 个朝下箭头,表示"水的重量压在底板上",旁标 `LOAD`。
8. **两侧文字**:水侧 `FLOOD SIDE`,墙右 `DRY SIDE`。
9. **页脚注**:`Principle — The flood's own weight presses the apron down — no fixing to the building.`
   (把物理叙事讲成一句真话,同时充当无障碍文本)

### 动态行为(克制,遵循动效治理)

- **只有水面动**:水面由**两条正弦波叠加**缓慢流动(一快一慢、方向相反),幅度极小(≈3px)、速度很慢——"轻呼吸"档。
- **挡水板、箭头、刻度、文字全部静止**。
- **`prefers-reduced-motion`**:水面**冻结**为静态曲线,信息零丢失。
- 无入场动画、无滚动触发——是一张"活着的仪器图",不是特效。

### 配色(用现有站 token,不要带独立稿那套色)

天然属于现有**诚实线描图家族**,描边走主题 token(`currentColor` / 语义变量),明暗自动适配:

| 元素 | 用色 |
| --- | --- |
| 水填充 | 很淡的蓝(低透明度) |
| 水面线 / 虚线 / LOAD 箭头 / FLOOD SIDE | 中蓝(可读水色) |
| 墙体 + 底板 | 主文字色(浅色=深墨 / 暗色=浅色,token 自动翻转) |
| 墙顶顶帽 + `TB-BW` | 现有站**单一强调色**(不要引入新色) |
| 刻度 / 网格 / 地面排线 / DRY SIDE | 中性(muted / line) |

> 原则:**水是"被描绘对象"用蓝,顶帽/标签这种"要强调的一点点"才用强调色**,强调色占比极小。

### 放在哪、多大

- **最合适**:**TB-BW(ABS Boxwall)产品页**顶部,作"工作原理"图——最相关、最诚实。
- 次选:首页 hero 旗舰视觉。
- 宽高比约 **4:3**;放进带页眉/页脚的卡片容器最佳。

### 实现提示(踩过的坑,务必转告实现方)

1. **技术选型**:**Canvas 2D** 最省事(水填充 + 波动曲线好画);SVG+CSS 也行但水波更费劲。
2. **必须 DPR 适配**(`devicePixelRatio`),否则高清屏发虚。
3. **⚠️ 尺寸竞态坑**:容器在字体/布局稳定前后尺寸会变,**只在初始化测一次会把图画偏、缩在角落**。
   解决:用 `ResizeObserver`,或**动画每帧重测容器尺寸、变了就重设画布**。
4. **无障碍**:canvas 加 `aria-hidden`,语义靠页脚那句真话承载,别把"原理"只画在图里。
5. **健壮性**:遵循"内容可见性不赌在 JS/动画上"——图是**增强性**的;JS 不跑时其余内容照常,必要时放静态首帧兜底。

## 资产二:页脚设计

### 概念

一个**发丝线结构、扁平、克制**的页脚:一个宽的品牌列 + 两个功能链接列,底部压一条
**法务/身份信息条**。没有色块、没有重底,靠版式、等宽微标签和细线说话——和"工程文档"气质一致。

### 结构(三列 + 底部法务条)

```
┌─ 顶部发丝线 ────────────────────────────────────────────────┐
│                                                              │
│  [品牌列 · 较宽]        [Navigation]        [Support]         │
│  TUCS=NBERG            Products            Request a Quote    │
│  一句业务描述          OEM & Wholesale     Contact           │
│  (工厂直供…)          Materials Guide     Warranty Policy    │
│                       About               Privacy · Terms    │
│                                                              │
├─ 发丝线 ─────────────────────────────────────────────────────┤
│  公司法务全称 · 主体公司 · sales@邮箱          © 2026 品牌     │
└──────────────────────────────────────────────────────────────┘
```

- **主网格**:3 列,品牌列更宽(约 `1.4fr` : `1fr` : `1fr`),列间距 ~32px;
  ≤720px 收成 2 列。
- **品牌列**:wordmark(带那个强调色的 `=` 标记)+ 一句业务描述(~13px、muted、限宽 ~34ch)。
- **链接列**:每列一个**等宽大写微标签**(Navigation / Support)——~11px、字距 0.1em、muted、
  weight 600;下面是**纯文本块状链接**——~14px、次要墨色、每条上下 padding ~4px、无项目符号、无下划线,hover 转强调/链接色。
- **法务条**:与网格之间加间距 + 一条发丝线隔开;**等宽 ~11.5px、muted**;
  左 = 公司法务全称 + 主体公司 + 销售邮箱,右 = 版权;两端对齐,移动端换行。

### 内容分组的逻辑(不是随便塞)

- **Navigation** = 发现类(逛产品、看指南、了解公司);
- **Support** = 转化 + 信任类(报价、联系、质保、隐私条款);
- **法务条** = 严肃 B2B 该有的身份背书(工商全称、主体、邮箱、版权)——买家判断"这家靠不靠谱"会看这条。

### 为什么好(要保留的气质)

- **纯发丝线结构 + 扁平**,和站点克制风格一致,不喧宾夺主;
- **等宽微标签**给技术/规格感,并把"栏目标题"和"链接"清晰分层;
- **两层信息**:功能链接目录 + 法务身份条,各司其职;
- **全页脚唯一的强调色只有 logo 的 `=` 标记**,其余全中性。

### 配色 / token

全部走现有站 token:顶部与法务条的分隔线用 `line`;链接默认次要墨色、hover 用链接/强调色;
微标签与法务条用 muted;`=` 标记用现有站强调色。**不引入独立稿的任何新色值。**

### 实现提示

- 纯 HTML + CSS(grid),无需 JS。
- 链接是块级、可点区域够大(每条含上下 padding),移动端点击友好。
- i18n:所有文案(栏目名、链接名、法务条)必须走翻译 key,不硬编码。
- 法务全称 / 邮箱 / 版权年份从现有站配置(如 `single-site` 品牌配置)取,别写死。

## 资产三:片数计算器(借鉴 NOAQ,改造为"报价漏斗")

> 参考:同行 NOAQ Boxwall 片数计算器 `https://www.noaq.com.tw/calculator.html`
>(分析于 2026-07)。**借鉴它的实用性,但要修掉它最大的漏洞。**

### 概念

一个**自助估算工具**:买家输入要防护的**长度**,即时算出**需要多少片/节/个**。
把"我到底要买多少"这个第一道摩擦,在买家自己手上一秒解决——降低门槛、教育买家、
筛出真实需求。

### NOAQ 原版拆解(它做了什么)

| 维度 | NOAQ 的做法 |
| --- | --- |
| 输入 | **单一"长度"**,可切换 公分/公尺;默认 0 |
| 型号 | 只有一个(Boxwall),无高度/型号选择 |
| 转角/端头/搭接 | **完全不问**,只按总长度算 |
| 单片有效面宽 | **90 公分** |
| 算法 | `片数 = 进位取整(输入长度cm ÷ 90)`(不足一片进位成一片) |
| 输出 | 片数(`N 片`)+ 一个可左右滑动的**拼装预览图** |
| 免责声明 | 「本系統僅依輸入長度自動估算所需片數,實際配置數量仍須依現場環境及防洪需求進行評估」 |
| 结果后的动作 | **无**——算完就完了,页脚只有通用联系方式,**没有任何转化按钮** |

### NOAQ 的致命漏洞 = 你们的反超点

**它算完什么都不做。** 买家好不容易算出"我要 12 片",然后……没有下一步。
一个高意向的动作被白白浪费。

> **你们的版本:结果 →「把这个配置发去报价」按钮,一键把长度/数量带进 RFQ 表单。**
> 把一个"实用小工具"变成**报价漏斗 / 获客引擎**。这和你们
>「不公开价格、报价即对话」的政策天然契合:**计算器只给数量,不给价;要价格就来报价**。

### 改造原则(务必遵守)

1. **只输出数量,永不输出价格**——片数/节数/袋数 + 需要的配件,绝不出现金额。
   与 `no-published-price` 政策一致。
2. **结果直连 RFQ**:主 CTA「Get a quote for this / 按此配置报价」,
   把已算出的**长度 + 数量 + 产品线**预填进现有报价表单(复用 `?interest=` 一类参数)。
3. **诚实声明,而且比 NOAQ 更诚实**:不只说"仅供估算",要**点名到底哪些因素会改变结果**
   ——转角、端头封边、搭接、地面平整度、是否需要排水。这正是你们品牌的诚实边界气质,
   顺带把"屏障不是排水方案"的老实话再讲一次。
4. **进位取整**:买不了半片,一律向上取整(同 NOAQ)。
5. **每条产品线的常数 = owner 真实规格,严禁编造**:NOAQ 的 90cm 是**它的**数,不是你们的。
   你们每条线的"单片/单节有效尺寸"必须来自真实规格表(spec 纪律 + 诚实)。见下方数据缺口。

### 适用哪几条线(别硬套所有线)

| 线 | 适用度 | 估算方式 |
| --- | --- | --- |
| **TB-BW ABS Boxwall** | ✅ 最适合 | `片数 = 进位(长度 ÷ 单片有效宽)`——和 NOAQ 同型 |
| **TB-TD 管式挡水坝** | ✅ 适合 | `节数 = 进位(长度 ÷ 单节有效长)` |
| **TB-FB 吸水袋** | ⚠️ 可做 | 按覆盖长度或点位估袋数(需 owner 定"每米/每点用几袋") |
| **TB-AG 铝制挡水闸** | ❌ 弱 | 按开口逐一定制报价,不是简单片数;顶多按"开口高度 ÷ 单板高"估板数,但仍需图纸 |
| **TB-CP FRP** | ❌ 不做 | 研发中 |

> MVP 建议:**先只做 TB-BW**(和 NOAQ 正面对标、最成熟),结构上预留可扩展到 TB-TD / TB-FB。

### 交互 / 布局

1. 选产品线(MVP 可固定 TB-BW)→ 2. 输入长度(cm/m 切换)→
3. **实时**显示所需片数(+ 可选:拼装示意 / 需要的端头件、转角件提示)→
4. 诚实声明 → 5. 主 CTA「按此配置报价」。
- 视觉走现有站 token 与组件规范,克制、发丝线结构,和剖面图属同一"工程"家族。

### 实现提示

- **纯客户端 JS** 即可算,无需后端;只是一个乘除 + 向上取整。
- **常数进配置**:单片有效尺寸放进产品/`single-site` 配置或产品规格数据,**不硬编码在组件里**。
- **i18n**:所有文案(标签、单位、声明、CTA)走翻译 key。
- **无障碍**:输入有 `<label>`;结果用 `aria-live="polite"` 播报,数字变化对读屏可感知。
- **接 RFQ**:CTA 携带 `产品线 + 长度 + 数量` 预填现有报价表单,复用现有 lead 管道。
- **健壮性**:JS 不跑时,该区块要么隐藏、要么退化成"联系我们报价"的静态入口,不留坏掉的空壳。

### 需要 owner 提供的数据(数据缺口)

- [ ] TB-BW 单片**有效防护宽度**(真实规格,cm)——对标 NOAQ 的 90cm
- [ ] TB-TD 单节**有效防护长度**(若做 TB-TD)
- [ ] TB-FB **每米/每点用量**(若做 TB-FB)
- [ ] 转角件 / 端头封边件是否单独计数、如何计(决定要不要做"进阶模式")
