# Grid Layout System

> Current Tucsenberg grid truth is a **hero-only decorative overlay**, not a
> site-wide editorial grid frame.

## Runtime surface

`HeroGuideOverlay` in `src/components/grid/hero-guide-overlay.tsx`:

- absolute, centered, `max-width: 1080px`
- `width: calc(100% - 48px)` (matches `px-6` gutters)
- class contract: `hidden lg:grid` (12×8 templates via style, **no** inline `display`)
- guide cells draw `border-right` / `border-bottom` with `--grid-guide`
- below `lg`: Tailwind `hidden` → `display: none`; from `lg`: `lg:grid` → `display: grid`
- `pointer-events: none` and `aria-hidden`

Do not set inline `display: grid` on this node — it overrides `hidden` and keeps
the overlay visible on mobile.

Guide coordinates come from `heroGuides()` in
`src/components/grid/hero-guides.ts`.

## Where it appears

Home hero (`src/components/sections/hero-section-view.tsx`) mounts
`<HeroGuideOverlay />` behind hero content.

There is **no** live whole-page decorative grid frame and no requirement to wrap whole
pages in a decorative grid.

## Specs

| Property | Value |
| --- | --- |
| Columns × rows | 12 × 8 |
| Content max width | 1080px |
| Horizontal inset | 24px each side (`48px` total) |
| Guide border | 1px `--grid-guide` |
| Visibility | `hidden lg:grid` (`lg+` only; no inline `display`) |

## Do not revive

- Whole-page decorative grid wrappers as a required page shell
- Treating guide lines as the primary layout mechanism for every section
- Copying retired starter grid essays as current implementation truth

For ordinary section layout, use the 1080px container and flex/grid utilities.
For page patterns, see `docs/design/页面模式.md`.
