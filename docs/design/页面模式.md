# Page Pattern Reference

> Current Tucsenberg marketing-page patterns after the 2026 redesign.
> This is a pattern reference for new work and refactors. Always verify the
> live route, tokens in `src/app/globals.css`, and component code before
> treating any example as mandatory.

---

## 1. Page skeleton

```
┌─ <html> ──────────────────────────────────────────────┐
│  Header (layout.tsx, ~56px nav)                        │
│  ┌─ <main> ──────────────────────────────────────────┐ │
│  │  Hero (optional HeroGuideOverlay on lg+)          │ │
│  │  ┌─ Section ─────────────────────────────────────┐│ │
│  │  │  Container (1080px + px-6)                    ││ │
│  │  │  └─ SectionHead / .text-section + Content     ││ │
│  │  └──────────────────────────────────────────────┘│ │
│  │  Final CTA section                                │ │
│  └───────────────────────────────────────────────────┘ │
│  Footer (neutral / light surface, not a dark band)     │
└────────────────────────────────────────────────────────┘
```

There is **no** live whole-page decorative grid frame. Decorative guide lines
exist only as the hero-local `HeroGuideOverlay`.

### Page shell

```tsx
<div className="min-h-screen bg-background text-foreground">{children}</div>
```

---

## 2. Container

```tsx
<div className="mx-auto max-w-[1080px] px-6">
```

| Property | Value |
| --- | --- |
| max-width | `1080px` |
| padding-x | `px-6` (24px) |

Narrow legal pages may use `max-w-[720px] px-6` with `mx-auto`.

---

## 3. Section rhythm

```tsx
<section className="py-14 md:py-[72px]">
```

Use `section-divider` when a top rule is needed between middle sections.

---

## 4. Type ladder

### H1 — page title (hero)

Use the live hero heading classes on each route. One H1 per page.

### H2 — section title (site-wide truth)

Canonical step is `.text-section`:

```text
mobile:  24px (text-2xl)
desktop: 28px
weight:  semibold
```

```tsx
<h2 className="text-section">...</h2>
```

`SectionHead` uses `.text-section`. Prefer `<SectionHead>` for reusable
title + subtitle + optional action layouts (contact FAQ, FAQ sections).

```tsx
<SectionHead title={title} subtitle={subtitle} />
```

Do **not** invent a second H2 track with hardcoded `text-[32px]`.

`.text-heading` (32 / md:36) is reserved for stronger display moments, not
ordinary section titles.

---

## 5. Cards

Cards use the **border system**, not drop-shadow chrome:

```tsx
<div className="surface-card p-6">...</div>
```

`.surface-card` = radius + `1px` border + card background
(`src/app/globals.css`). The shared `Card` primitive applies `surface-card`.

Do not document “no borders, use shadow only” — that is pre-redesign advice.

---

## 6. Buttons

Heights come from tokens:

| Size | Token | Computed |
| --- | --- | --- |
| default | `--button-height-default` | **40px** (`2.5rem`) |
| lg | `--button-height-lg` | **48px** (`3rem`) |
| sm | `--button-height-sm` | 32px |

Use `Button` from `src/components/ui/button.tsx` / `button-variants.ts`.

---

## 7. Footer

Footer is a **neutral light surface** in the current theme:

| Token | Role |
| --- | --- |
| `--footer-bg` | `var(--neutral-1)` (light mode) |
| `--footer-text` | `var(--neutral-10)` |
| `--footer-heading` | `var(--neutral-11)` |
| `--footer-link` | `var(--neutral-9)` |

Do not recreate the retired dark slate footer band.

Implementation: `src/components/footer/Footer.tsx`.

---

## 8. Decorative grid

Only `HeroGuideOverlay` (`src/components/grid/hero-guide-overlay.tsx`):

- 12×8 guides, max width 1080px
- `lg+` only, pointer-events none
- Used inside the home hero, not as a whole-page frame

See `docs/design/网格系统.md`.

---

## Checklist for new marketing sections

- [ ] Container is `max-w-[1080px] px-6`
- [ ] Section H2 uses `.text-section` or `SectionHead`
- [ ] Cards use `.surface-card` / `Card`
- [ ] Primary/secondary CTAs use shared `Button` sizes (40 / 48)
- [ ] No new page-wide grid frame
- [ ] Footer stays on neutral tokens
