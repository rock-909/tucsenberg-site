# Integrations

这份文档记录后续可能引入的插件、组件和集成清单。  
它是决策清单，不是调研资料堆。

## 当前使用规则

只有当某个候选能明确解决当前问题，或者能显著提升后续复用性时，才推进接入。

## 候选清单

| 名称 | 类型 | 当前状态 | 解决的问题 | 现在是否推进 | 备注 |
|---|---|---|---|---|---|
| 更多 shadcn/ui 组件 | 组件库扩展 | 可按需引入 | 加快表单、弹层、导航等交互拼装 | 按需 | 当前项目已经在用 shadcn/ui，不需要预先批量安装 |
| 更强的动效/交互组件 | 设计增强 | 观察中 | 某些页面需要更强视觉表现时可复用 | 暂不默认推进 | 先以当前设计系统和 CSS/Tailwind 动效为主 |
| 更丰富的图库/灯箱组件 | 内容展示组件 | 观察中 | 产品图、服务图、案例图后续可能需要更好的查看体验 | 暂不默认推进 | 当前高质量图片资产不足，先不急着引入 |
| 更细的分析/归因集成 | 数据集成 | 观察中 | 后续若更重视转化归因和页面行为分析，可增强事件链路 | 暂不默认推进 | 先把现有主转化链路打稳 |
| SEO/Schema 辅助组件 | SEO 辅助 | 观察中 | 当内容规模扩大后，帮助统一结构化数据和 SEO 输出 | 暂不默认推进 | 当前先保持手写和现有 helper 主导 |

## 更新规则

- 新候选先加到这里，再决定是否接入
- 已经明确不做的候选，也保留一行结论，避免重复讨论
