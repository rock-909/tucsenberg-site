# i18n 设置

这个项目使用 `next-intl` 做多语言。这里的重点不是重组代码，而是让新项目接手时知道：多语言到底改哪里、按什么顺序改、哪些地方不能乱引用。

## 多语言真相分三层

| 层级 | 负责什么 | 主要位置 |
| --- | --- | --- |
| 路由 locale | 网站支持哪些语言、默认语言、locale 路由规则、语言展示名、时区和货币 | `src/config/paths/locales-config.ts`、`src/i18n/routing-config.ts`、`src/i18n/routing.ts`、`src/i18n/request.ts`、`src/i18n/locale-utils.ts` |
| messages JSON | 按钮、表单、导航、提示语这类界面文案 | `messages/{locale}/critical.json`、`messages/{locale}/deferred.json` |
| MDX 页面内容 | 页面正文、页面 SEO、每个 locale 的页面资料 | `content/pages/{locale}/*.mdx` |

简单说：

- 路由决定“有哪些语言入口”。
- messages 决定“界面零散文案怎么翻译”。
- MDX 决定“页面正文和 SEO 内容是什么”。

## 主真相表

| 文件或目录 | 用途 |
| --- | --- |
| `src/config/paths/locales-config.ts` | 真正的 locale 配置主真相：locale 列表、默认 locale、prefix、展示名、时区和货币。 |
| `src/i18n/routing-config.ts` | 把 locale 配置接入 `next-intl` 的 `defineRouting(...)`，是 middleware / runtime 边界可用的 edge-safe routing config。 |
| `src/i18n/routing.ts` | App 页面和组件使用的 `next-intl` navigation helpers，不是 edge-safe 入口。 |
| `src/i18n/request.ts` | 每次请求如何加载当前 locale 的 messages、timezone 和 formats。 |
| `src/i18n/locale-utils.ts` | locale 判断、规范化和辅助工具。 |
| `messages/{locale}/critical.json` | 首屏和关键交互需要的界面文案。 |
| `messages/{locale}/deferred.json` | 非首屏或可延后加载的界面文案。 |
| `content/pages/{locale}/*.mdx` | 每个语言版本的页面正文、页面级 metadata 和 SEO 内容。 |

如果只是判断哪些 message namespace 需要替换，先看 `message-namespace-map.md`。如果讨论是否要把 `critical.json` / `deferred.json` 再拆细，先看 `messages-split-feasibility.md`；当前决策是不拆文件、不改 i18n runtime。

## 修改顺序

新增、删除或调整语言时，按这个顺序来：

1. 先改 `src/config/paths/locales-config.ts`，确认 locale 列表、默认语言、prefix、展示名、时区和货币。
2. 再确认 `src/i18n/routing-config.ts` 仍然正确消费这份 locale 配置。
3. 再补齐 `messages/{locale}/critical.json` 和 `messages/{locale}/deferred.json`。
4. 再补齐 `content/pages/{locale}/*.mdx`。
5. 最后检查路由、message contract 和内容检查。

不要反过来先复制页面内容。页面文件存在，不代表这个语言已经被路由、messages 和检查链完整支持。

## 边界

- `middleware` 或其他 edge-sensitive 文件不要 import `src/i18n/routing.ts`。
- App 页面和组件需要 navigation helpers 时，使用 `src/i18n/routing.ts`。
- 用户可见文案不要硬编码在组件里，要放进对应的 messages JSON 或 MDX。
- 页面正文和页面 SEO 属于 MDX，不属于 translation JSON。
- translation JSON 更适合放按钮、导航、表单字段、校验提示、状态提示这类界面文案。

## 验证命令

改完 i18n 相关内容后，至少跑：

```bash
pnpm content:check
pnpm exec vitest run src/i18n/__tests__/routing.test.ts src/i18n/__tests__/request.test.ts tests/unit/i18n.test.ts tests/unit/i18n-message-contract.test.ts
```
