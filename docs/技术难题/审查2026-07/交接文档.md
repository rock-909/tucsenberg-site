## 3. 关键操作经验（新会话务必知道）

- **分支命名必须 `pr/NN-*`**：`.github/workflows/ci.yml` 只在 `pull_request: branches: [main, "pr/**"]`（+ push 到 main）触发 CI。堆叠 PR 的 base 若是 `audit/*` 之类 → **完全不跑 CI**（"no checks reported"）。所以每个堆叠分支都要 `pr/NN-...`，PR1 base=main，之后每个 base=上一个 `pr/NN`。
- **commitlint 要求 subject 全小写**：不能有任何大写。写 `link-href`/`locale`，不要 `LinkHref`/`Locale`。
- **提交前必须 `pnpm exec prettier --write <改动文件>`**，否则 format-check 卡住。
- **eslint `--max-warnings 0`**：多余的 `eslint-disable` 指令是 warning，会挡提交。测试里动态 `fs` 读需 `// eslint-disable-next-line security/detect-non-literal-fs-filename -- <理由>`（`readFileSync(变量)` 需要，`readdirSync("字面量")` 不需要）。
- **本会话 IDE/LSP 诊断完全不可信**：反复报幻影的 "Cannot find module"、"toBeInTheDocument does not exist"、"Unterminated regex"、错误行号（引用比文件还长的行）。**唯一权威是 `pnpm type-check`（tsc --noEmit）+ `pnpm test` + CI**。不要因为 LSP 红就改代码，先跑 type-check。
- **审查报告只在 pr/00 分支**：`docs/技术难题/审查2026-07/findings-*.md`（13 批 + `findings-cross.md` 主 backlog）+ 本交接文档，都在 pr/00-audit-docs（#40）。代码栈 pr/01–pr/10 是从 main 切的，**不含**这些文档。子 agent 在代码分支上读不到，需靠内联 scope 或先 checkout pr/00。
- **删除用 `git rm`**（本次授权的死代码清理，可经 git 历史恢复，分支未合并）。全局规则"移到 Trash"针对的是用户数据/未跟踪文件；被授权清理的已跟踪源码用 git rm 是对的。
- **持久化教训**：会话临时目录 `/private/tmp/.../scratchpad` 曾被系统清理一次，丢了 5 份报告（后从子 agent transcript 恢复）。**耐久产物立即入库 git，别放会话临时目录。**
- **owner 工作流规矩**：每个 PR 搞定 → 提交 → 推送 → 开 PR → CI 全绿 → **停在合并前**。合并永远是 owner 的。
