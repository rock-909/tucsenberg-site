# 证据日志

## 1. 证据口径

本轮只接受以下证据进入最终结论：

1. 目标提交上的新鲜命令、测试、构建、运行时或浏览器结果。
2. 目标提交当前 source/config/content 的直接静态证据。
3. 绑定目标 SHA 的 GitHub Actions 运行。
4. 经过主线程推翻式反证后仍成立的单源 finding，或由两条独立车道分别报告的同一根因。

旧审查、计划、README 和未绑定 SHA 的公开 preview 只用于找线索或说明边界，不作为 P0/P1 的决定性证据。

六个服务证明等级保持分开：浏览器发出请求、route 返回成功、mock 被调用、真实服务接受、真实邮件或记录存在、owner 实际收到。后一级不能由前一级替代。
