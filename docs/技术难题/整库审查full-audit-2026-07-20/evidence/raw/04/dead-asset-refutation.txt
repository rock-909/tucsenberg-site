# Next.js public-folder semantics (installed 16.2.10 docs)

     6	
     7	{/* The content of this doc is shared between the app and pages router. You can use the `<PagesOnly>Content</PagesOnly>` component to add content that is specific to the Pages Router. Any shared content should not be wrapped in a component. */}
     8	
     9	Next.js can serve static files, like images, under a folder called `public` in the root directory. Files inside `public` can then be referenced by your code starting from the base URL (`/`).
    10	
    11	For example, the file `public/avatars/me.png` can be viewed by visiting the `/avatars/me.png` path. The code to display that image might look like:
    12	
    13	```jsx filename="avatar.js"
    14	import Image from 'next/image'
    15	
    16	export function Avatar({ id, alt }) {
    17	  return <Image src={`/avatars/${id}.png`} alt={alt} width="64" height="64" />
    18	}
    19	
    20	export function AvatarOfMe() {
    21	  return <Avatar id="me" alt="A portrait of me" />
    22	}
    23	```
    24	
    25	## Caching
    26	
    27	Next.js cannot safely cache assets in the `public` folder because they may change. The default caching headers applied are:
    28	
    29	```jsx
    30	Cache-Control: public, max-age=0
    31	```
