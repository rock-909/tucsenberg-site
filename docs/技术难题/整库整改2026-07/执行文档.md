## 第一批:真实业务损失 + 假绿灯

### PR-1 lead 响应关键路径(`pr/fix-lead-response-path`)

**问题**:政策是 email-first-storage-optional,但 `src/lib/lead-pipeline/process-lead.ts:186,287` 用 `Promise.all` 同步等齐邮件与 Airtable;Airtable SDK 未配超时,默认 `requestTimeout=300000`。买家可能白等 5 分钟。

**改法**:
1. 给 Airtable 客户端设置明确的短请求预算(`requestTimeout` 选项;量级秒级,执行者给出取值依据)。
2. 响应成功条件对齐政策:contact/inquiry 通道邮件成功即可返回成功,Airtable 结果降级记录(日志/错误码),不阻塞响应。注意 subscribe 通道是 Airtable-only(`process-lead.ts:312`),成功条件不同,单独处理。
3. 新增行为测试:"邮件成功、Airtable 永不返回 → 响应在预算内成功"。
4. 若响应语义变化,同步 `行为合约.md`(BC-012A 相关)与 lead-family 契约测试。

**验证**:`pnpm test`(lead-pipeline、lead-family 全套)+ `pnpm build`。
