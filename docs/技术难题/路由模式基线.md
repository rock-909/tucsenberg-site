# Route Mode Baseline

This is a proof note, not a hard gate.

## Expected shape

Fresh `pnpm build` on the current runtime tip should report:

```text
○ static: /_not-found, /api/health, icons, robots.txt, sitemap.xml
◐ Partial Prerender: localized pages and localized catch-all
ƒ dynamic: /api/inquiry and /api/csp-report
```

When investigating route mode drift, read the current `pnpm build` route
summary directly from the build log.

Keep `src/middleware.ts` as the current runtime entrypoint.
Do not introduce `src/proxy.ts` just to silence a framework warning. <!-- truth-docs:allow-missing -->
Cloudflare/OpenNext support is not acceptable for a blind migration. Proxy migration is a separate runtime lane; see `../项目基础/技术栈.md`.

## Known warning

`pnpm build` may emit non-blocking
`BAILOUT_TO_CLIENT_SIDE_RENDERING: useSearchParams()` logs from
`src/components/navigation/navigation-progress-bar.tsx:82`. The build still exits
successfully. If the warning recurs or spreads beyond that component, investigate
from fresh build output rather than assuming an older localized-404 attribution.

Do not describe this warning as fixed in C7, and do not change the component in
this documentation-only lane.
