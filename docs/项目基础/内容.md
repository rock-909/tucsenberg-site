# Content

这里说明派生项目怎么替换页面正文、SEO、messages 和多语言。文件级索引看 `替换边界.md`，message ownership 看 `消息文案.md`。

static public pages are registered in `src/config/pages.config.ts`; page content uses `content/pages/{locale}` and generated compatibility messages use `messages/{locale}`.

CMS / operator editing boundary: the default model is content-as-code. If a
CMS-like editor is added, it should first act as a thin Git-backed editing layer
over approved MDX page files, not as a replacement source of truth for the whole
site. See `../决策记录/Content-as-code与CMS边界.md`.

## What to replace

- 内容行为和验证选项：`content/config/content.json`
- 页面正文和页面级 SEO：`content/pages/{locale}/*.mdx`
- UI 文案、按钮、导航、表单提示：`messages/base/**` 和 `messages/profiles/**`
- 兼容输出：`messages/{locale}/critical.json`、`messages/{locale}/deferred.json`
- 页面组合和 CTA：`src/config/single-site-page-expression.ts`
- SEO/crawl/sitemap：`src/config/single-site-seo.ts`

默认 `company-site` 必看 Home、About、Products overview、Blog、Resources、Contact、Privacy、Terms。`/products/[market]`、Capabilities、How it works、Custom project support 只属于 optional profile 或显式选择的 surface。

## MDX rules

- `title` / `description` 是页面内容输入，不等于最终浏览器 title。
- `seo.title` / `seo.description` 是 metadata 输入。
- `updatedAt` 表示买家能看到的内容有变化。
- `lastReviewed` 表示人工复核时间，法务和联系页尤其要维护。
- 不要把页面长正文塞进 messages JSON。

## i18n rules

多语言真相分三层：

| Layer | Main files |
| --- | --- |
| locale routing | `src/config/paths/locales-config.ts`, `src/i18n/routing-config.ts`, `src/i18n/request.ts` |
| UI messages | `messages/base/{locale}/**`, `messages/profiles/{profile}/{locale}/**` |
| page content | `content/pages/{locale}/*.mdx` |

新增或删除语言时，先改 locale config，再补 message packs 和 MDX。不要只复制页面文件。

改 message pack 后同步 compat：

```bash
tsx scripts/starter-profile/sync-message-compat.ts --write
```

Compat 新鲜度由 `pnpm content:check` 校验。

Historical split notes from the starter were not retained in this derived
checkout. Current message ownership lives in `消息文案.md`.

## Proof

```bash
pnpm content:check
node scripts/starter-checks.js translations
node scripts/starter-checks.js content-readiness --profile company-site
```

如果改 locale runtime，再加：

```bash
pnpm exec vitest run src/i18n/__tests__/routing.test.ts src/i18n/__tests__/request.test.ts tests/unit/i18n-message-contract.test.ts
```
