## Runtime model

- App Router 是页面主框架。
- Route Handlers 是公开写入主通道；浏览器联系表单走 `/api/contact`。
- Server Actions 保留为兼容入口，不是浏览器联系表单主路径。
- 内容、翻译、页面表达更新默认通过重新构建和重新部署发布。
- 当前不依赖 R2 / D1 / Durable Object 作为内容缓存或失效机制。
- `cacheComponents: true` 已启用；`cacheTag()` / runtime tag invalidation 不是上线架构的一部分。

## Cloudflare / OpenNext

常用命令：

```bash
pnpm build
pnpm website:build:cf
pnpm exec wrangler deploy --dry-run --env preview
node scripts/starter-checks.js deployed-smoke --base-url <url>
```

规则：

- `pnpm build` 和 `pnpm website:build:cf` 不要并行跑。
- `pnpm website:build:cf` 通过不等于页面可访问。
- Wrangler dry-run 只证明 artifact 可走到 dry-run 阶段，不证明真实上线。
- 真实上线还需要 deployed smoke、表单 canary 和 owner signoff。
- `DEPLOYMENT_PLATFORM=cloudflare` 是 canonical 平台信号；`DEPLOY_TARGET=cloudflare` 是 legacy compatibility alias。

Cloudflare 问题先分桶：

1. platform entry / local runtime issue；
2. generated artifact compatibility issue；
3. current site runtime regression；
4. final deployed behavior issue。

## Compatibility date

`wrangler.jsonc` 当前使用：

- `compatibility_date: 2026-05-04`
- `compatibility_flags: ["nodejs_compat", "global_fetch_strictly_public"]`

`compatibility_date` 是 Workers runtime 行为开关，不跟随依赖补丁自动前进。改它至少跑：

```bash
pnpm build
pnpm website:build:cf
```

涉及上线或部署行为时再补 Wrangler dry-run / preview smoke。

## Dependency upgrade policy

- 安全补丁优先，不能用 hold 掩盖漏洞。
- patch / minor 可以小批量升级，但要跑对应证明。
- major、`0.x` minor、编译器、格式化器、部署工具大版本变更要单独开 lane。
- `@types/node` 跟 Node 24 runtime 对齐，不升到 Node 25 types，除非 runtime baseline 先扩到 Node 25。
- TypeScript / ESLint / React Email / OpenNext / Wrangler 升级不能只改版本号；要跑类型、lint、build、Cloudflare build 和相关 focused tests。
- `react-email` 当前保持 6.1.1；`@react-email/ui` 已到 6.6.0。仓库当前对 `chokidar@4.0.3` 保留一个窄的 `trustPolicyExclude`，因为 pnpm 会把它判成 trust downgrade，但 `react-email` 4.x/5.x/6.x 线都持续依赖 `^4.0.3`。这个例外只放行精确版本 `chokidar@4.0.3`，不放宽全局 `trustPolicy: no-downgrade`。
- `pnpm` 的根配置以 `pnpm-workspace.yaml` 为准；不要继续往 `package.json` 的 `pnpm` 字段写 overrides / peer rules / build allowlist。仓库当前把 `verifyDepsBeforeRun` 设为 `true`，让依赖状态不一致时在脚本运行前阻断；先显式运行 `pnpm install` 再继续验证。

## CSP / proxy decisions

- Decision: do not implement nonce CSP now.
- Current default remains static-compatible CSP.
- No middleware/proxy migration test in this lane.
- nonce CSP future proof must cover dynamic rendering, proxy-generated nonce, Cache Components, Cloudflare/OpenNext, Turnstile, analytics, deployed-smoke, real-service-canary, and rollback plan.
- official-doc-only check: Decision: do not migrate middleware.ts to proxy.ts now.
- src/middleware.ts remains the runtime entrypoint; src/proxy.ts is not created.
- No runtime migration test in this lane.
- Next.js docs checked for this decision: node_modules/next/dist/docs/01-app/03-api-reference/03-file-conventions/proxy.md, node_modules/next/dist/docs/01-app/01-getting-started/16-proxy.md, node_modules/next/dist/docs/01-app/03-api-reference/03-file-conventions/02-route-segment-config/runtime.md, node_modules/next/dist/docs/01-app/02-guides/content-security-policy.md.
- 默认保持 static-compatible CSP。
- `NEXT_PUBLIC_SECURITY_MODE=strict` 表示启用当前静态兼容安全 headers，不表示 nonce CSP。
- nonce CSP 需要单独证明动态渲染、nonce 传递、Cloudflare/OpenNext、Turnstile、analytics、浏览器行为和回滚方案。
- 当前不把 `src/middleware.ts` 盲目改成 `src/proxy.ts`。Proxy 迁移必须是独立 runtime lane，不能和 CSP、i18n、Cloudflare 或文档清理混在一起。

## Known debt

- `script-src-elem 'unsafe-inline'` 是 static App Router/RSC bootstrap 折中；没有用户生成富文本时风险较低。
- Localized 404 prerender 可能输出已知 `DYNAMIC_SERVER_USAGE` 日志；如果扩散到正常页面再按回归处理。
- Local Cloudflare preview 比 deployed proof 弱；冲突时以 deployed runtime evidence 优先。

如果旧长文或历史 proof 被恢复到 `docs/archive/**`，只作追溯材料，不作当前技术真相。
