# 文档清单

这份文件说明 `docs/` 里的每个文档为什么还存在。它不是产品真相入口；当前工作从 `../README.md` 开始。

## 生命周期标签

| 标签 | 含义 |
| --- | --- |
| `current-entry` | 当前维护入口或目录入口。 |
| `current-reference` | 仍在维护的规则、边界、设计真相或配置说明。 |
| `current-proof` | 当前验证、发布或质量门证明。 |
| `inherited-starter-reference` | 仍需保留的上游 starter 参考，不代表当前产品真相。 |
| `historical-proof` | 历史证明或旧阶段证据，不代表当前上线状态。 |
| `method-workflow` | 可复用方法或过程记录，本身不是当前产品真相。 |
| `candidate-backlog` | 候选想法或未来可能接入的事项，不是已批准工作。 |

Docs existence closeout: 每个 tracked `docs/` 文件都已经有保留理由。以后新增或移动文档时，同一轮更新这里。

## 目录

| 目录 | 作用 | 处理 |
| --- | --- | --- |
| `docs/` | 项目知识库入口。 | 只保留 `README.md` 和主题目录。 |
| `docs/项目基础/` | 技术栈、配置、内容、部署、验证和继承边界。 | 当前维护优先从这里读。 |
| `docs/design/` | 设计真相、组件治理、Storybook 和视觉系统。 | 设计类规则留在这里。 |
| `docs/技术难题/` | 性能、Cloudflare、Next.js、路由和客户端边界等问题记录。 | 记录问题、证据和方法，不当作发布入口。 |
| `docs/决策记录/` | 已确认的架构或产品技术决策。 | 保留 ADR。 |
| `docs/audits/` | 修复前审查和计划记录。 | `historical-proof`；只用于追溯。 |
| `docs/superpowers/` | Superpowers 上游输出根目录。 | `historical-proof`；`specs/**`、`plans/**` 保留原始决策与执行记录，不是当前产品真相。 |
