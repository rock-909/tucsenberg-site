# Surfaces

This is the file-level replacement index. Use `替换顺序.md` for the step-by-step flow.

## Labels

- `must-replace`: replace or confirm before client launch.
- `review-or-tune`: starter structure may stay, but review for the selected project.
- `do-not-edit-first`: facade/generated/runtime mechanism; change upstream truth instead.

## Root tooling surfaces

Some root files support local collaboration and review. They are not buyer
content or runtime configuration:

- `conductor.json`
- `conductor-setup.sh`
- `.coderabbit.yaml`
- `skills-lock.json`

- `.mcp.example.json`: safe developer integration example; replace or remove it
  if the site does not use MCP tooling.
- `semgrep.yml`: site security scan config used by `.github/workflows/**`. It
  is current while the Semgrep CI job exists; remove or rewrite both together
  if the project chooses another security scan.

## Catalog truth

The site is permanently the single English catalog site. Current Tucsenberg
product detail pages and their JSON-LD use
`src/constants/tucsenberg-product-page-*.ts`, assembled by
`src/constants/tucsenberg-product-pages.ts`, as runtime truth.
`src/config/single-site-product-catalog.ts` and
`src/constants/product-standards.ts` provide the route/list/catalog adapters.

## Do not edit first

- `src/config/paths/site-config.ts`: runtime/validation facade; edit `src/config/single-site.ts` first.
- `src/constants/product-catalog.ts`: query facade; edit current product page truth in `src/constants/tucsenberg-product-pages.ts` or route/list adapters in `src/config/single-site-product-catalog.ts` and `src/constants/product-standards.ts` first.
- `src/lib/content-manifest.generated.ts`: generated.
- `src/lib/mdx-importers.generated.ts`: generated.
- `src/app/**/page.tsx`: route owner; do not hardcode brand, product facts, or long page copy there.

Regenerate content artifacts with:

```bash
node scripts/starter-checks.js content-manifest
node scripts/starter-checks.js content-manifest --check
```

Minimum default replacement proof:

```bash
pnpm brand:check
pnpm content:check
node scripts/starter-checks.js translations
node scripts/starter-checks.js content-readiness
pnpm exec vitest run tests/architecture/website-config-runtime-boundary.test.ts
```
