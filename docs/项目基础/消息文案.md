# Messages

Physical message packs are the authoring truth. Files under
`messages/{locale}/` are generated compatibility output.

## Fixed graph

The Tucsenberg site always composes messages in this order:

```text
base -> b2b-lead -> catalog
```

The directories are:

```text
messages/base/{locale}/{critical,deferred}.json
messages/profiles/b2b-lead/{locale}/{critical,deferred}.json
messages/profiles/catalog/{locale}/{critical,deferred}.json
```

`b2b-lead` and `catalog` are fixed ownership layers. They are not selectable
runtime profiles.

## Generated compatibility files

Do not hand-edit `messages/en/critical.json` or
`messages/en/deferred.json`. After changing a physical pack, run:

```bash
pnpm messages:sync
pnpm content:check
```

`pnpm messages:sync` deep-merges the fixed graph and writes the compatibility
files. `pnpm content:check` verifies that the generated files still match the
physical packs.

## Ownership

| Surface | Owner |
| --- | --- |
| shared shell, legal, errors, accessibility, theme | `messages/base/**` |
| contact, RFQ, lead form, trust, email helper copy | `messages/profiles/b2b-lead/**` |
| catalog, products, product navigation and product-page UI | `messages/profiles/catalog/**` |
| page prose and page SEO | `content/pages/en/*.mdx` |
| product specifications and product copy | `src/constants/tucsenberg-product-page-*.ts` |

Long page prose does not belong in message JSON. Product specifications do not
belong in message packs.

## Transactional email copy

Website messages and transactional email templates are related but separate:

- next-intl reads the website message graph;
- the lead pipeline renders React Email templates;
- `emailTemplates` in the base deferred pack is the authoring/reference
  namespace for transactional email copy;
- `src/emails/email-copy.ts` is the runtime email copy API used by templates
  and subject helpers.

runtime email rendering still defaults to English, matching the current site.
