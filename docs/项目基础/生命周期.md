# Lifecycle

This repository is now one concrete Tucsenberg website. Runtime profile
selection, profile materialization, demo fixtures, and the starter blog source
are retired. Historical derivation records remain only to explain earlier
decisions.

## Current lifecycle

Use `site-long-term` for code, docs, checks, and deployment proof that maintain
the English catalog site:

- type-check, lint, tests, and production build;
- content, messages, and generated manifest checks;
- form, API, Turnstile, rate-limit, and lead-delivery contracts;
- component governance and Storybook proof;
- Cloudflare/OpenNext build and deployed smoke proof.

The fixed message ownership graph is `base -> b2b-lead -> catalog`. Validate
packs with `pnpm content:check`.

## Check lifecycle

| Check | Current role |
| --- | --- |
| `pnpm type-check`, `pnpm lint:check`, `pnpm test`, `pnpm build` | Core local and CI proof. |
| `pnpm content:check` | Manifest, slug, translation pack shape, and key-usage gates. |
| `node scripts/starter-checks.js content-readiness` | Catalog-only buyer-visible placeholder scan. |
| `node scripts/starter-checks.js truth-docs` | Stable docs and historical-boundary gate. |
| `pnpm component:check` | Component governance tests, scan, and Storybook build. |
| `pnpm website:build:cf` | Cloudflare/OpenNext artifact proof. |
| deployed smoke and real lead canary | Manual launch proof. |

Current commands must not depend on historical derivation records. If a
historical file conflicts with runtime code, current configuration and fresh
proof win.
