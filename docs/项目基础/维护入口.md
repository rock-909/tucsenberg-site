# 维护入口

这里放当前 Tucsenberg 站维护说明，以及已经标成 Historical 的派生记录。

当前站已经派生完成。日常维护不要从 `派生起步.md` 开始；先看内容、部署和项目边界。

## 阅读顺序

1. `内容.md`：页面正文、SEO、messages 和内容检查。
2. `部署.md`：Cloudflare、env、表单和上线前配置。
3. `monitoring.md`：Cookie、GA4、Search Console、Airtable 和流量监控阶段动作。
4. `品牌.md`：公司身份、域名、联系方式、品牌资产。
5. `AI协作边界.md`：Codex / Claude 协作边界。
6. `替换顺序.md`：继承的 starter 替换面说明；只在追溯时看。
7. `派生起步.md`：已退役 derivation flow 的历史说明；不是当前站日常入口。
8. 旧 starter workflow 说明已经移出 `docs/`；当前 docs 不再保留 workflow 目录。

## 清理规则

当前 runtime 不保留 profile selection、materialization 或 profile
fixtures。`b2b-lead` 和 `catalog` message directories 是固定 ownership
layers，不是可选 profile。

清理前仍要确认 CI、脚本、测试、docs 和 runtime 引用。当前 message 机制看
`消息文案.md`；旧派生机制只在带 Historical banner 的记录中查看。
